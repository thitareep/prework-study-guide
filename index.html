HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element defines the title for the webpage.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewpoint.
  • The script element embeds client-side scripts in HTML.
  • Child elements are elements nested inside other elements.
  • Parent element is an element that holds children elements.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS is used when wanting to style an element directly in the HTML file.
  • Internal style sheet contains CSS rules for the webpage in the head section.
  • External CSS style sheet is preferred since it keeps CSS rules in a separate file and makes design changes easier.
  • The selector defines the elements or attributes to which the rules or declarations will apply.
  • Declarations contain the CSS property we want to apply and teh value of the property.
  • Assigning colors can be done in semantic references or referred to color by it's hexadecimal code (hex #).

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Repository (Repo) is a folder in which all files and their version histories are stored.
  • A branch is a workspace which you can make changes that won't affect the live site.
  • Commit Changes is a saved record of a change made to a file within the repo.
  • Pull request (PR) is the way to ask for changes made to a branch to be merged into another branch.
  • A pull request also allows for multiple users to see, discuss, and review work being done.
  • Merge is when a pull request is approved and the commit is pulled in from one branch to another, then deployed on the live site.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JavaScript is case-sensitive.
  • Conditionals are code structures to test if an expression returns true or not.
  • A function is a set of instructions that tells the computer how to perform a certain task.
  • Event handlers are code structures that listen for activity in the browser and run code in response.
  • JavaScript language consists of primitive values and objects.
  • Loops contain 3 statements: the starting point for the loop, the condition, and the final statement allows the array iterate over each item.
  • Requirements for a JavaScript function: Define the function and call the function.
  • An array is a single variable that issued to hold a group of data.
